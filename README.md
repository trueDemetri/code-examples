# Примеры кода

Здесь выложил код некоторых инструментов собственной разработки (это часть моей библиотеки DTools), а также пример использования одного из них.

## Messenger
Это легковесная реализации системы обмена сигналами по принципу SignalBus (полный код). Использовал такую в Heroic Squad.
В проекте можно использовать много экземпляров для разных уровней. Например,
- ProjectMessenger (сигналы верхнего уровня между большими системами в проекте)
- LevelMessenger (сигналы между системами сцены уровня)
- UnitMessenger (сигналы между внутренними системами боевого юнита)
- и т.д.

Сигналы лучше реализовывать в виде структур для избежания лишних аллокаций памяти. Интерфейс применяется для защиты от ситуации отправки сигналов не в том контексте (например, чтобы не передать сигнал проекта в LevelMessenger). Для этого сигналы одного уровня делаем с одним пустым флаговым интерфейсом.

```c#
public interface IProjectSignal
{}

public class ProjectMessenger : Messenger<IProjectSignal>
{}

public readonly struct SceneLoadedSignal : IProjectSignal
{
  public readonly string SceneName;

  public SceneLoadedSignal(string sceneName)
  {
    SceneName = sceneName;
  }
}

//...

ProjectMessenger messenger;
//...
// Нет аллокации. Этосокращенная запись для messenger.Fire<SceneLoadedSignal>(new SceneLoadedSignal("Level1"))
messenger.Fire(new SceneLoadedSignal("Level1"))
```

Также если в проект добавить библиотеку Jetbrains Lifetimes и включить флаг компиляции DTOOLS_LIFETIMES, то включается поддержка Lifetime для указания времени жизни подписки на сигнал.

## WeaponFSM

Пример использования описанной ваше машины состояний для реализация простого функционала оружия.

Само оружие представлено классом Weapon, которое через интерфейс даёт наружу свои данные в виде реактивных свойств (ViewableProperty из Jetbrains Lifetimes) только для чтения. Они позволяют View-системам легко реагировать на все изменения модели оружия.

Внутри же с помощью механизма FSM происходит работа оружия (стрельба очередями, перезарядка, корректное реагирование на запросы в разных состояниях).


В общем, код подобрал несложный, в нём всё видно.
